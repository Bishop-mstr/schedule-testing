# Чеклісти для тестування
## Модуль: Управління парами групи (Варіант 4)

### Функціональність (Позитивні сценарії)
- [X] Створення пари з усіма заповненими полями (Предмет, Викладач, Тип, Години).
- [X] Редагування існуючої пари: зміна викладача на іншого зі списку.
- [X] Додавання валідного посилання (URL) для дистанційного заняття.
- [X] Автоматичне копіювання назви з поля "Subject" у поле "Subject for site" при першому введенні.

### Негативні сценарії
- [X] Спроба зберегти пару з порожнім полем "Subject" (очікується помилка).
- [X] Спроба зберегти пару без вибору викладача (очікується помилка).
- [X] Введення 0 або від’ємної кількості годин.
- [X] Введення екстремально великої кількості годин.
- [X] Введення тексту замість цифр у поле "Hours".
- [X] Введення некоректного посилання (без http/https).
- [X] Спроба зберегти пару без вибору жодної групи.

### UI/UX та візуальна перевірка
- [X] Наявність візуальних позначок (наприклад, *) біля обов'язкових для заповнення полів.
- [X] Коректність відображення довгих назв предметів у випадаючому списку.
- [X] Наявність повідомлення про успішне збереження після натискання кнопки "Save".
- [X] Перевірка, чи кнопка "Зберегти" стає неактивною під час процесу відправки даних, щоб уникнути дублікатів.
