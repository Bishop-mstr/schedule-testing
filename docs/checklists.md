# Чеклісти для тестування
## Модуль: Управління парами групи (Варіант 4)

### Функціональність (Позитивні сценарії)
- [ ] Створення пари з усіма заповненими полями (Предмет, Викладач, Тип, Години).
- [ ] Редагування існуючої пари: зміна викладача на іншого зі списку.
- [ ] Створення "спільної пари" шляхом вибору кількох груп одночасно.
- [ ] Додавання валідного посилання (URL) для дистанційного заняття.
- [ ] Автоматичне копіювання назви з поля "Subject" у поле "Subject for site" при першому введенні.

### Негативні сценарії
- [ ] Спроба зберегти пару з порожнім полем "Subject" (очікується помилка).
- [ ] Спроба зберегти пару без вибору викладача (очікується помилка).
- [ ] Введення 0 або від’ємної кількості годин (наприклад, -1).
- [ ] Введення екстремально великої кількості годин (наприклад, 999 або твої 69).
- [ ] Введення тексту замість цифр у поле "Hours".
- [ ] Введення некоректного посилання (без http/https) у поле лінка.
- [ ] Спроба зберегти пару без вибору жодної групи.

### UI/UX та візуальна перевірка
- [X] Наявність візуальних позначок (наприклад, *) біля обов'язкових для заповнення полів.
- [ ] Коректність відображення довгих назв предметів у випадаючому списку (чи не обрізається текст).
- [ ] Наявність повідомлення про успішне збереження після натискання кнопки "Save".
- [ ] Перевірка, чи кнопка "Save" стає неактивною під час процесу відправки даних, щоб уникнути дублікатів.
